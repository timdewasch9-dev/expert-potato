# Architecture Portfolio

A minimalist, monochrome portfolio website — plain HTML, CSS, and JavaScript.
No build tools, no frameworks. Just open the files or host them anywhere.

The design follows the conventions of leading architecture portfolios
(John Pawson, Studio MK27, Snøhetta): near-invisible navigation, full-bleed
black-and-white photography that turns to colour on hover, generous whitespace,
and hairline rules that echo architectural drafting.

---

## Editing your content

**You only need to edit one file:** [`js/data.js`](js/data.js)

It controls everything — your name, bio, contact details, and every project.
Open it in any text editor; it is heavily commented.

### Your details
At the top, `window.SITE` holds your name, role, tagline, email, phone,
location, and social links. The `about` object holds the About page content:
`intro` paragraphs, plus `education`, `experience`, `skills`, `other`, and
`languages` lists that render the CV section automatically.

### Adding / editing projects
Inside `window.PROJECTS`, each `{ ... }` block is one project. To add a project,
copy a block, paste it, and edit the fields:

| Field         | What it is                                            |
|---------------|-------------------------------------------------------|
| `slug`        | A unique id for the URL (lowercase, dashes, no spaces)|
| `title`       | Project name                                          |
| `location`    | e.g. "Oslo, Norway"                                   |
| `year`        | e.g. "2025"                                           |
| `category`    | e.g. "Residential", "Cultural"                        |
| `cover`       | The grid/thumbnail image                              |
| `description` | One quoted string per paragraph                       |
| `facts`       | Key/value pairs shown on the project page             |
| `images`      | List of images for the project gallery                |

The homepage grid and the individual project pages update automatically.

## Adding your images — no code editing needed

The site already points at `.jpg` files for every project. Until a `.jpg`
exists, the matching grey **labelled placeholder** is shown automatically
(each label tells you which drawing/render belongs there). So all you do is
**upload an image with the matching name** and it appears — no editing.

You can do this entirely on GitHub in your browser:

1. In the repo, open the **`assets/img`** folder.
2. Click **Add file → Upload files**, drag your images in, **Commit changes**.
3. Refresh the site — your image replaces the placeholder.

Name each file exactly as listed below (keep the `.jpg`). The number after the
name is the order it appears in that project's gallery; `-cover` is the
homepage thumbnail.

| Project          | Files to upload                                            |
|------------------|------------------------------------------------------------|
| mise en space    | `mise-en-space-cover.jpg`, `mise-en-space-1.jpg` … `-6.jpg`|
| soft parasite    | `soft-parasite-cover.jpg`, `soft-parasite-1.jpg` … `-4.jpg`|
| resonant growth  | `resonant-growth-cover.jpg`, `resonant-growth-1.jpg` … `-5.jpg`|
| raw'25           | `raw-25-cover.jpg`, `raw-25-1.jpg` … `-4.jpg`              |
| hexenhaus 2.0    | `hexenhaus-2-cover.jpg`, `hexenhaus-2-1.jpg` … `-4.jpg`    |
| reculture        | `reculture-cover.jpg`, `reculture-1.jpg`, `reculture-2.jpg`|
| through the wall | `through-the-wall-cover.jpg`, `through-the-wall-1.jpg` … `-4.jpg`|
| (about portrait) | `portrait.jpg`                                             |

Tips:
- `.png` works too — just change the extension in `js/data.js` to match.
- Want more or fewer images in a project? Add/remove lines in that project's
  `images` list in `js/data.js`.
- Resize large photos to ~2000px on the long edge so the site loads fast.

The labelled placeholders are generated by `scripts/gen-placeholders.js`;
you can ignore or delete them once your real photos are in.

## Previewing locally

Easiest: **double-click `index.html`** to open it in your browser.

Or, for a more accurate preview, run a tiny local server from this folder:

```bash
python3 -m http.server 8000
# then open http://localhost:8000
```

## Publishing (free)

- **GitHub Pages:** push this repo, then in GitHub → Settings → Pages, set the
  source to your branch root. Your site goes live at
  `https://<username>.github.io/<repo>/`.
- **Netlify / Vercel:** drag-and-drop this folder, or connect the repo. No build
  command needed — it's a static site.
